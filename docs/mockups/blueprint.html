Narrative Summary REF: 002

BudgetWise started as a weekend experiment to scratch my own itch: I wanted a finance tracker that understood developer spending patterns. Most budgeting apps treat all subscriptions equally, but I needed something that could differentiate between a $20/month SaaS tool that pays for itself in productivity and a streaming service I forgot to cancel.

The project grew from a simple CSV parser into a full-stack application with AI-powered transaction categorization. The core insight was that spending patterns are more useful than spending totals. Instead of showing a pie chart of where money went, BudgetWise surfaces trends and anomalies that need attention.

The hardest part was not the ML model or the data pipeline. It was designing an interface that makes financial data feel approachable without dumbing it down. Every chart, every number, every color choice went through multiple iterations to find the right balance between information density and clarity.

Key Decisions REF: 006
[1]TypeScript over JavaScript

[2]Prisma for schema management
Auto-generated types from the database schema eliminated an entire category of runtime errors and kept the API layer honest.
[3]Rule engine over hardcoded categories
Built a composable rule system so users can define their own categorization logic instead of relying on a fixed taxonomy.
