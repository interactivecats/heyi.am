BudgetWise

github.com/alex/budgetwise budgetwise.app
Screenshot placeholder — 900 × 384

BudgetWise started as a weekend experiment to replace a spreadsheet that had grown unwieldy over three years of manual transaction categorization. The core insight was that most personal finance apps either do too much (full accounting suite) or too little (simple expense tracker with no analysis).

I wanted something in between: automatic categorization using a lightweight ML model trained on my own spending patterns, paired with a dashboard that surfaces trends instead of burying them in drill-down menus. The AI categorizer runs locally using ONNX Runtime, so no transaction data ever leaves the device.

The most challenging piece was building the reconciliation engine that handles duplicate detection across multiple bank imports. After three iterations, I landed on a fuzzy-matching approach using Levenshtein distance on normalized merchant names combined with amount-date proximity scoring.

Key Decisions
  • Prisma over Drizzle for schema management
    Prisma's migration tooling and type generation made schema iteration faster during rapid prototyping. Trade-off: heavier runtime dependency.
  • Stripe Connect for payment processing
    Chose Stripe over building custom payment handling to avoid PCI compliance burden. Connect enables multi-account support out of the box.
  • PostgreSQL tsvector for search
    Full-text search via tsvector beats external search service for this scale. Built custom ranking with ts_rank_cd and trigram similarity fallback.
  • CSV import with streaming parser
    Papa Parse streaming mode handles 100k+ row imports without memory pressure. Progress callback drives the upload UI bar.
