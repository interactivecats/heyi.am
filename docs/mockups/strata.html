BudgetWise started as a weekend experiment to replace the spreadsheet I'd been using for years. The core idea was simple: envelope budgeting shouldn't require a finance degree. Every dollar gets assigned a job, and the UI makes it obvious when you're overspending.

The interesting engineering challenge was recurring transaction detection. I built a fuzzy matcher that identifies patterns in bank imports -- same merchant, similar amounts, regular intervals -- and suggests automations. It reduced manual categorization by about 70% in my own usage.

The hardest part wasn't the algorithms. It was making the onboarding flow feel approachable. Three failed iterations before landing on the guided budget builder that walks you through your first month.

Key Decisions

Envelope budgeting over zero-based
Zero-based requires allocating every dollar upfront. Envelopes let users start with the categories they care about and expand over time. Lower friction for new users.
Client-side CSV parsing
Bank statements never leave the browser. Parsing happens in a Web Worker to keep the UI responsive. Privacy-first -- no server-side storage of raw financial data.
PostgreSQL over SQLite for multi-device
Started with SQLite for simplicity but migrated to PostgreSQL when adding multi-device sync. The schema migration was painful but necessary for real-time collaboration.

Project Phases

Phase 1: Foundation
Set up Prisma schema, configured PostgreSQL, established project structure with TypeScript. Built the core data model for accounts, envelopes, and transactions.
Phase 2: Transaction Engine
Implemented CSV import with client-side parsing, built the fuzzy matcher for recurring transaction detection, and created the categorization UI with drag-and-drop envelope assignment.
Phase 3: Insights and Search
Built the spending insights dashboard with trend charts, added full-text search using tsvector for transaction history, and integrated Stripe for premium features.
Phase 4: Polish and Ship
Mobile responsive layout, two-factor authentication, accessibility audit, and performance optimization. Reduced initial bundle from 340KB to 180KB gzipped.
