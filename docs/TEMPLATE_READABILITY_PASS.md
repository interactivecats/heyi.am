## Architecture (critical — easy to get wrong)

- **Source of truth**: `cli/src/render/templates/{name}/styles.css` + shared `cli/src/render/templates/styles.css`
- **Auto-synced target**: `heyi_am_umbrella/apps/heyi_am_public_web/priv/static/css/templates/` — regenerated by `scripts/sync-phoenix-assets.sh`, which the pre-commit hook runs. **Never hand-edit.**
- **Phoenix's own CSS**: `heyi_am_umbrella/apps/heyi_am_public_web/assets/css/app.css` — page-shell for the portfolio list page, NOT synced from CLI. Edit directly.
- **Design reference only**: `docs/mockups/{name}/project.html` — not rendered at runtime, can drift. Update only when useful as reference.

## Remaining work — detail

### /adapt
These templates lack mobile breakpoints or have only a single one, audit surfaced:
- mono, paper, chalk, glacier, aurora, verdant

Add `@media (max-width: 768px)` that shrinks type, collapses grids, relaxes padding. Cross-check `docs/TEMPLATE_FIX_LEARNINGS.md` for per-template responsive notes.

### /animate
These already have `prefers-reduced-motion` blocks (verified during /harden):
- parallax (line ~727), cosmos (line ~193), kinetic (line ~477), showcase (line ~695) — all have both CSS `@media` and JS `window.matchMedia` paths.

Audit remaining templates with animations (scan for `@keyframes` or `animation:`). Any template that uses an animation but lacks a reduced-motion guard needs one. The pattern from parallax is the best template.

### /polish
- `exec-path-number` in Phoenix `app.css:982` is 32px — non-interactive (verified), so WCAG 2.5.5 doesn't require 44px. But if visually small, bump to ~2.25rem here.
- Final consistency sweep once /normalize + /adapt land.
